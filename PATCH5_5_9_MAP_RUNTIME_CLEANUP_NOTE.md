# PATCH 5.5.9 — MapRuntimeCleanup

기준 버전: `MapPatch5_5_8_1_MapLayerVisibilityHotfix`

## 목적

5.5 계열 지도 패치가 누적되면서 줌/드래그 입력 하나에 여러 패치의 HUD, LOD, 클러스터, QA 갱신이 동시에 반응하는 문제를 줄인다.

이번 패치는 새 시각효과를 추가하지 않고, 지도 런타임 입력 경로를 정리한다.

## 수정 내용

### 1. 문서 캡처 단계 지도 입력 통합

지도 줌 버튼, 휠 줌, 드래그 입력을 document capture 단계에서 먼저 처리한다.

- 오래된 shell/frame 단위 줌 리스너가 중복 반응하는 것 완화
- 지도 조작은 `MapRuntimeCleanup` 경로에서만 처리
- 일반 마커 클릭, 지역 탭, 필터 버튼은 기존 동작 유지

### 2. 줌 단계 커밋 지연

줌 중에는 transform과 readout만 즉시 갱신하고, `data-zoom-stage` / `data-pc55-stage`는 입력이 멈춘 뒤 한 번만 갱신한다.

이로 인해 다음 작업이 매 프레임 반복되지 않도록 한다.

- 클러스터 재계산
- LOD visibility rule 적용
- FEEDS 패널 갱신
- QA 패널 갱신
- tactical visual refresh

### 3. display:none 반복 핫픽스 방식 사용하지 않음

5.5.6.1에서 문제를 만들었던 조작 중 `display:none`/복구 반복 방식을 쓰지 않는다.

이번 패치는 조작 중 장식 레이어의 transition/animation만 낮추고, DOM 구조는 유지한다.

### 4. 검수 패널은 디버그 모드에서만 표시

일반 모드에서 QA 패널들이 화면과 렌더링 부담을 늘리지 않도록 숨김을 강화한다.

## 건드리지 않은 것

- 지도 마커 데이터
- 좌표 투영 데이터
- 지도 레이어 시각 디자인
- 세력관계도
- 기록보관서
- F.H.C / 영상 / 개체 연출
- 전체 효과음 믹싱

## 검수 포인트

1. 줌인/줌아웃 시 이전보다 덜 끊기는지
2. 휠 줌 시 HUD/레이어가 과하게 반복 갱신되지 않는지
3. 드래그 중 프레임 밀림이 줄었는지
4. 조작 종료 후 레이어/클러스터/상세 패널이 정상 갱신되는지
5. 마커 클릭과 클러스터 선택 목록이 정상 작동하는지
6. 지역 탭/필터 버튼이 정상 작동하는지
