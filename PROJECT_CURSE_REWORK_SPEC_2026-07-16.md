# Project Curse 전면 개편 기준서

- 기준 패키지: `5.15.2cd Internal Structure Stabilization Pass`
- 작성일: 2026-07-16
- 상태: 구현 전 확정 기준
- 다음 구현 패스: `5.15.2ce Organization Canon Reconciliation Pass`

이 문서는 현재 사이트를 계속 덧붙이는 방식에서 벗어나, 설정 원본과 화면 표현을 분리한 새 구조로 개편하기 위한 단일 기준서다. 아래 다섯 표는 구현 중 임의 해석을 막는 승인 기준으로 사용한다.

1. 화면 상태표
2. 효과음 사건표
3. 문서 작성 규칙
4. 공식 용어집
5. 보호·검수표

## 0. 개편 범위와 절대 제한

### 목표

- 태그 버튼, 기록 수 칩, 일반 연결 패널처럼 정보를 반복하는 UI를 줄인다.
- 연표·지도·관계도·세력 파일·기록보관소가 같은 사건 데이터를 공유하게 한다.
- 흩어진 설정글을 대표 기록철 내부의 절과 증거 자료로 합친다.
- 메뉴 전환, 기록 열람, 미디어 재생의 상태와 음향을 한 곳에서 관리한다.
- 한국어 본문을 우선하고 영어는 공식명·파일 ID·상태 코드에만 제한한다.
- 기존 화면을 끝에 덧붙인 패치로 보정하지 않고, 새 소유 파일로 분리한다.

### 절대 보호 대상

다음 두 기록은 완성본이다.

- `종교` — `Cults_871104`
- `불멸을 향해` — `Immortality_860201`

보호 범위는 본문, 기록면, 이미지, 영상, 효과, 음향, 레이아웃, 반응형 동작 전체다. 독립 문서뿐 아니라 `index.html` 안의 인라인 원본도 포함한다. 공통 CSS나 공통 오디오 변경으로 간접적인 차이가 생기는 것도 실패로 처리한다.

### 이번 개편에서 하지 않는 것

- 두 잠금 기록의 내용 보강 또는 문장 교정
- 우시노다교 설정을 잠금 기록 안에 새로 삽입
- 확인되지 않은 조직의 영문 풀네임 생성
- 장식용 태그와 새 기록 카드를 무작정 늘리는 작업
- 모든 메뉴에 긴 가짜 로딩을 넣는 작업
- 모든 클릭·호버에 소리를 붙이는 작업
- 실제 근거 없이 세력의 소속·동맹·적대 관계를 확정하는 작업

# 1. 화면 상태표

## 1.1 전역 상태

| 상태 | 진입 조건 | 화면 동작 | 음향 동작 | 종료 조건 |
|---|---|---|---|---|
| `BOOT` | 첫 접속 | 시스템 상태와 현재 보관소만 1.5~2초 이내 표시 | 사용자 입력 전 자동 재생 금지 | 첫 입력 또는 부팅 완료 |
| `MENU_IDLE` | 여섯 기본 화면 열람 | 현재 화면을 완전히 표시하고 입력 허용 | 메뉴 환경음이 켜져 있으면 낮은 음량으로 유지 | 화면 이동·기록 열람·탭 비활성화 |
| `MENU_TRANSITION` | 다른 기본 화면 선택 | 이전 화면 감쇠 → 얇은 스캔 → 새 제목 → 내용 순서로 0.4~0.6초 전환 | 전환음 1회, 연속 입력 잠금 | 새 화면 준비 완료 |
| `RECORD_LOADING` | 기록 열기 | ID·권한·매체 상태를 0.14~0.30초 표시 | 기록 장착음 1회, 환경음 감쇠 | 기록 본문 준비 또는 접근 거부 |
| `RECORD_VIEW` | 기록 준비 완료 | 본문·증거·관련 사건을 표시 | 일반 UI음 최소화 | 뒤로가기·다른 기록·미디어 재생 |
| `MEDIA_PLAYING` | 영상·음성 증거 재생 | 해당 매체에 시각적 우선권 부여 | 메뉴 환경음 중지 또는 덕킹, 기록 전용 음향 우선 | 재생 종료·정지·기록 이탈 |
| `RECORD_EXITING` | 기록에서 목록으로 복귀 | 기록면을 닫고 목록의 이전 위치 복원 | 짧은 잠금 해제음 1회 | 목록 복원 완료 |
| `ACCESS_DENIED` | 봉인·권한 없음 | 화면 이동 없이 이유와 등급만 짧게 표시 | 거부음 1회, 추가 클릭 쿨다운 | 안내 종료 |
| `HIDDEN` | 브라우저 탭 비활성화 | 애니메이션과 불필요한 갱신 중지 | 모든 공통 음향 일시정지 | 탭 복귀 |

### 상태 규칙

- 한 시점에 전역 상태는 하나만 활성화한다.
- `MENU_TRANSITION` 중 입력은 마지막 정상 선택 하나만 보존한다.
- 실제 데이터 준비가 끝났다면 로딩 연출을 억지로 늘리지 않는다.
- 이미 열어본 화면은 약식 전환을 사용한다.
- 뒤로가기는 새 화면을 만드는 것이 아니라 이전 선택·스크롤·필터 상태를 복원한다.
- 잠금 기록은 기존 전용 진입·재생 상태를 그대로 사용하며 새 상태 관리자가 내부 동작을 가로채지 않는다.

## 1.2 화면별 정보와 등장 순서

| 화면 ID | 화면 목적 | 기본 등장 순서 | 선택 시 연동 대상 | 제거·축소 대상 |
|---|---|---|---|---|
| `terminal-home` | 단말의 현재 기록·경보·최근 사건 진입점 | 상태 → 최근 기록 → 활성 사건 → 접근 경고 | 기록·사건 | 장식용 상태 칩, 중복 안내문 |
| `history` | 1980~2030 사건 흐름 | 시간축 → 연대 → 사건 → 현재 영향 | 지도 좌표·관여 세력·관련 기록 | 동일 사건을 다시 설명하는 장문 카드 |
| `region-map` | 실제 지리 기반 상황 파악 | 기본 지도 → 관할·위험 구역 → 사건 표식 → 선택 정보 | 연표·세력·기록 | 대륙 모양만 있는 가상 도형, 상단 태그 줄 |
| `faction-relation` | 선택 세력을 중심으로 관계와 근거 확인 | 중심 세력 → 산하 → 협력·지원 → 외부·적대 → 연결 근거 | 세력 파일·사건·기록 | 별도 `연결 기록` 박스, 모든 관계를 같은 선으로 표시 |
| `faction-info` | 세력 하나의 정식 기록철 | 식별 → 목적 → 지휘 → 활동 → 한계 → 관계 → 관련 사건 | 관계도·지도·기록 | 작은 정보 박스 반복, 역할 목록 중복 |
| `archive-entry` | 모든 원본 기록의 검색·열람 | 검색 → 4개 서가 → 기록철 → 내부 절·증거 | 사건·세력·지도 | 태그 버튼 행, 기록 수 칩, 중복 파생 기록 |

## 1.3 공통 사건 선택 구조

모든 화면은 `selectedEventId` 하나를 공유한다. 화면별로 같은 사건을 다시 작성하지 않는다.

| 사건 필드 | 사용 화면 |
|---|---|
| `id`, `title`, `dateStart`, `dateEnd` | 연표·기록보관소 |
| `location`, `coordinates`, `regionId` | 지도 |
| `participants`, `relationsAtTime` | 관계도·세력 파일 |
| `recordIds`, `evidenceIds` | 기록보관소 |
| `summary`, `outcome`, `unresolved` | 연표·기록 상세 |
| `confidence`, `sourceRecordIds` | 관계선·추정 표기 |

예: `피의 호수 사건`을 선택하면 연표는 1986년으로 이동하고, 지도는 해당 좌표를 강조하며, 관계도는 당시 관여 세력만 활성화하고, 기록보관소는 연결된 원본 기록을 보여준다.

## 2.1 채널

## 2.3 금지 규칙

- 호버 효과음은 사용하지 않는다.
- 하나의 클릭에서 접점음·래치음·페이지음이 연쇄 재생되지 않게 한다.
- `new Audio()`를 각 버튼 처리기에서 직접 만들지 않는다.
- 화면별 코드가 음량을 임의 변경하지 않는다.
- 같은 효과음 이름을 서로 다른 파일로 재정의하지 않는다.
- 잠금 기록의 기존 전용 음향 파일과 재생 순서는 변경하지 않는다.
- 좌하단 고정 사운드 조절 UI를 다시 만들지 않는다. 필요하면 사이드 메뉴 안에 작은 설정 한 줄만 두고 선택을 기억한다.

## 2.4 현재 자산 처리 원칙

`5.15.2cd`의 `contact`, `latch`, `mount`, `projector`, `denied`, `boot` 자산은 후보 음원으로 유지할 수 있다. 다만 새 구현에서는 파일명이 아니라 위 사건명을 호출하고, 중앙 관리자가 실제 파일·볼륨·쿨다운을 결정한다.

# 3. 문서 작성 규칙

## 3.1 세 가지 문체

| 문체 | 사용 위치 | 문장 방식 | 예시 |
|---|---|---|---|
| 시스템 문체 | 상태·경고·버튼·시간 로그 | 명사형 또는 짧은 절차형 | `기록 복구 중`, `재개봉 금지`, `신호 소실` |
| 공식 기록문 | 세력·사건·개체·장비 문서 | 관찰·조치·한계를 분리한 평서문 | `외피가 착용자의 움직임보다 늦게 접히면 장비 전체를 봉인한다.` |
| 회수 발화문 | 교신·인터뷰·개인 메모 | 인물의 어휘와 불완전성을 유지 | 원문 화자의 말투를 보존하되 설명문처럼 다듬지 않음 |

## 3.2 문단 규칙

- 한 문단은 2~4문장으로 제한한다.
- 한 문단은 하나의 기능만 수행한다.
- 기본 순서는 `정의 → 관측·행동 → 예외·한계`다.
- 조직명을 주어로 반복하며 같은 정의를 되풀이하지 않는다.
- 임무 목록과 본문이 같은 내용을 반복하면 목록을 없애고 본문에 통합한다.
- 긴 기록은 새 카드로 쪼개지 않고 내부 제목·접힌 증거·타임스탬프로 나눈다.

### 수정 예시

기존형:

> U.A.C 정보 수집 부서는 도시 전역의 감시 영상, 통신 기록, 오디오 파일, 암호화 CCTV, 현장 데이터를 수집하고 복구하는 정보 부서이다.

권장형:

> U.A.C 정보수집부는 도시 감시망에서 영상·통신·생체 신호를 회수한다. 손상 자료는 사건 시각과 좌표를 기준으로 복구하며, 원본과 일치하지 않는 구간은 추정 정보로 분리한다.

## 3.3 문서 종류별 구조

| 종류 | 고정 순서 |
|---|---|
| 세력·기관 | 식별 → 목적 → 지휘 구조 → 활동 → 운용 한계 → 관계 → 관련 사건 |
| 인물 | 식별 → 확인된 이력 → 능력·행동 → 관련 사건 → 현재 평가 |
| 이상현상·개체 | 식별 → 취급 절차 → 관측 특성 → 발생 조건 → 사례 → 미확인 사항 |
| 사건 | 요약 → 시간순 경과 → 피해·손실 → 회수 증거 → 분석 → 미해결 사항 |
| 작전 매뉴얼 | 적용 범위 → 진입 조건 → 절차 → 중단 기준 → 철수 → 사후 처리 |
| 장비·기술 | 목적 → 운용자 → 사용 조건 → 성능 저하 → 중지·회수·폐기 |
| 세계 역사 | 짧은 배경 → 핵심 사건 → 조직 변화 → 현재에 남은 영향 |

## 3.4 증거 수준과 종결형

| 증거 수준 | 권장 종결형 |
|---|---|
| 확정 | `확인되었다`, `회수되었다`, `종료되었다` |
| 반복 관측 | `관측되었다`, `동일 반응을 보였다` |
| 분석·추론 | `~로 추정된다`, `~로 분류된다` |
| 가능성 | `가능성을 배제할 수 없다` |
| 미확인 | `원인은 확인되지 않았다`, `소재는 확인되지 않았다` |
| 현재 특성 | `~을 보인다`, `~가 발생한다` |
| 절차 | `~시 즉시 철수한다`, `~후 봉인한다` |
| 금지 | `~을 금한다`, `재개봉하지 않는다` |
| 시간 로그 | `진입`, `교신 두절`, `2명 미복귀`처럼 종결어미 생략 |

## 3.5 피해야 할 표현

| 피할 표현 | 교체 기준 |
|---|---|
| `강력한`, `엄청난`, `기괴한` | 실제 피해·크기·반응·관측 결과 |
| `관련되어 있다`, `영향을 준다` | 누가 무엇을 제공·차단·변조·회수했는지 명시 |
| `처리한다`, `관리한다` | 봉인·소각·분류·이관·격리 등 실제 동사 |
| `알려져 있다` | 정보 출처 또는 불확실성 표시 |
| 불필요한 영어 상태문 | 한국어 우선, 파일 ID·공식 약어만 유지 |
| 과도한 검열 블록 | 생존·취급 절차는 가리지 않음 |

## 3.6 기록보관소 통합 기준

- 핵심 기록철 목표 수: 약 12~15개.
- 서가: `사건`, `세력·인물`, `이상현상·개체`, `작전·장비·규정`의 4개.
- 기존의 흩어진 기록이 같은 사건·인물·장비를 설명하면 대표 기록철 하나로 합친다.
- `연결 기록`은 새 문서로 만들지 않고 원본 데이터의 `recordIds`, `eventIds`, `relationIds`로 계산한다.
- 부록은 본문 해석을 바꾸거나 새로운 증거를 제공할 때만 추가한다.
- 잠금 기록 두 개는 통합 대상이 아니며 독립 기록 그대로 유지한다.
