## PATCH5_8_1R_24_REGIONAL_MAP_MODULE_REBUILD_NOTE.md

# PATCH 5.8.1R-24 — RegionalMapModuleRebuild

## 목적

R19~R23에서 드러난 지역지도 필터/라벨/렌더링 충돌을 더 이상 후순위 덮어쓰기 방식으로 처리하지 않고, 지역지도 모듈만 새로 작성한다.

전체 사이트, 문서, 기록 보관소, 세력 정보, 세력 관계도, 오디오 리소스는 유지한다. 변경 범위는 `지역 지도` 기록면의 지도 모듈에 한정한다.

## 핵심 변경

### 1. 지역지도 모듈 재작성

기존 누적 패치가 사용하던 다음 계열과 분리된 새 모듈을 추가했다.

- 기존 `.continental-map-shell`
- 기존 `.continent-filter`
- 기존 `.continent-panel`
- 기존 `[data-map-item]`
- 기존 지도 MutationObserver 기반 후처리 패치

새 모듈은 아래 단일 구조로 작동한다.

- 단일 데이터: `DATA`
- 단일 권역 정의: `REGIONS`
- 단일 필터 정의: `FILTERS`
- 단일 렌더러: `r24_regional_map_rebuild.js`
- 단일 상태: `region / filter / review / selected`

### 2. 필터 시스템 완전 재작성

필터는 더 이상 기존 HTML/JS 여러 군데에서 생성하거나 재정렬하지 않는다. 새 모듈 내부의 `FILTERS` 배열 하나만 기준으로 만든다.

고정 필터:

- 전체
- 오염 구역
- 레드존
- 옐로우존
- 그린존
- 화이트존
- 블랙존
- 작전 시설
- 현상 기록
- 사건 좌표
- 봉쇄 거점
- 봉쇄선
- 검수 모드

검수 모드는 데이터 필터가 아니라 `reviewMode` 상태로 분리했다. 따라서 검수 모드를 눌러도 현재 필터가 망가지지 않는다.

### 3. 코드 라벨 처리 변경

기존 방식처럼 코드 라벨을 항상 만들고 CSS로 숨기지 않는다.

- 일반 화면: 내부 코드 라벨 DOM을 만들지 않음
- 검수 모드: 내부 코드 라벨 생성
- 검수 모드 해제: 다시 렌더링하면서 코드 라벨 제거

대상 예시:

- `WZ-*`
- `YZ-*`
- `GZ-*`
- `RZ-*`
- `BZ-*`
- `INC-*`
- `OBS-*`
- `GATE-*`
- `COM-*`
- `BL-*`

### 4. 세계지도 재작성

세계지도는 권역 선택 전용 화면으로 유지한다.

- 존/시설/사건/봉쇄선 데이터 레이어 없음
- 줌 없음
- 드래그 없음
- 필터 없음
- 권역 버튼만 표시
- 권역 버튼 클릭 시 상세지도 전환

### 5. 대륙 상세지도 재작성

각 상세지도는 새 렌더러가 SVG로 직접 생성한다.

지원 요소:

- 존
- 작전 시설
- 현상 기록
- 사건 좌표
- 봉쇄 거점
- 봉쇄선
- 통신권

표현 기준:

- 같은 종류라도 하나의 고정 표현에 묶지 않음
- 존 크기 다양화
- 봉쇄선/감시선 길이, 방향, 곡률 다양화
- 사건/현상/통신권 규모 차등
- 시설/기지 아이콘은 과하게 키우지 않고 가독성 중심으로 유지

### 6. 관계 기반 배치 유지

배치 기준은 R20에서 정리한 관계 기반 클러스터를 유지한다.

- 관련 있는 요소는 가까이 배치
- 관련 없는 요소는 분리
- 레드존 주변에는 관측기지, 사건, 현상, 봉쇄선 배치
- 봉쇄선 근처에는 봉쇄 거점 배치
- 항만 화이트존 근처에는 검문소 배치
- 블랙존 후보는 관련 레드존 심부/인접권에 배치
- 그린존은 후방 운영권으로 분리

## 수정 파일

- `index.html`
  - R24 새 지도 모듈 스크립트 연결
- `assets/js/r24_regional_map_rebuild.js`
  - 새 지역지도 모듈 추가
- `assets/css/style.css`
  - R24 지도 모듈 전용 스타일 추가
- `README.md`
  - R24 기준 설명 갱신
- `PATCH5_8_1R_24_REGIONAL_MAP_MODULE_REBUILD_NOTE.md`
  - 본 패치노트 추가

## 주의 사항

기존 지도 관련 누적 패치 파일을 물리적으로 전부 삭제하지는 않았다. 다른 문서/기록면과 연결된 런타임 영향 가능성을 줄이기 위해 보존하되, 지역지도 화면에서는 R24 모듈이 기존 지도 셸을 제거하고 새 UI를 렌더링한다.

향후 지역지도 수정은 기존 누적 패치에 덧붙이지 말고 `assets/js/r24_regional_map_rebuild.js`의 데이터와 렌더러를 기준으로 진행해야 한다.

## PATCH5_8_1R_25_REGIONAL_MAP_VISUAL_QA_PASS_NOTE.md

# PATCH 5.8.1R-25 — RegionalMapVisualQAPass

## 목적

R24에서 재작성한 지역지도 모듈을 실제 화면 기준으로 검수하고, 세계 권역 선택 화면·대륙 상세지도·필터·검수 모드·존/봉쇄선/마커 표현을 미세조정한다.

## 주요 수정

### 1. 세계 권역 선택 화면 보정

- 세계지도는 계속 권역 선택 화면으로 유지.
- 세계지도에 데이터 레이어를 추가하지 않음.
- 권역 버튼 위치를 실제 대륙 중심에 더 붙도록 소폭 조정.
- 세계지도 표시감과 크롭을 미세조정.

### 2. 봉쇄선/감시선 렌더링 개선

- 봉쇄선과 감시선을 꺾은 `polyline`이 아니라 곡선형 `path`로 렌더링.
- 해상 감시선, 항로 검문선, 사막 감시선, 항만 차단선의 곡률 차이를 더 자연스럽게 보정.
- `scale` 값을 실제 선 두께에 반영.

### 3. 관계권 강조

- 지도 요소를 선택하면 같은 `group`에 속한 요소를 은은하게 강조.
- 레드존-관측기지-봉쇄선-봉쇄 거점 같은 관계형 클러스터가 더 잘 보이도록 함.
- 관계 없는 요소는 강조하지 않음.

### 4. 존 톤 보정

- 블랙존의 붉은 경고판 느낌을 낮추고 심부 코어 느낌을 강화.
- 옐로우존은 감시/완충권처럼 보이도록 채도와 존재감을 낮춤.
- 그린존/화이트존은 후방 운영권/검문 관리권으로 보이도록 과한 오염감을 줄임.

### 5. 검수 모드 안정화

- 검수 모드는 기존 필터와 분리된 보기 상태로 유지.
- 일반 화면에서 내부 코드 라벨은 계속 숨김.
- 검수 모드에서만 내부 코드 라벨 표시.

## 유지 사항

- 새 존 체계 추가 없음.
- 세계지도 데이터 마커 추가 없음.
- 기존 문서/기록/세력 관계도 변경 없음.
- QA 폴더 복구 없음.

- R27 관계권 보정 데이터 유지

## 주요 변경점

### 1. 세계 권역 선택 화면 마감

- 권역 버튼 위치를 대륙 중심에 더 맞게 소폭 보정했습니다.

### 2. 존 표현 시각 보정

- 사막 감시선은 시설선처럼 보이지 않도록 흐리고 긴 감시선 톤으로 조정했습니다.

### 4. 마커 가독성 정리

# PATCH 5.8.1R-35 — TERMINAL NAVIGATION POLISH

## 목적
R31~R34에서 만든 지도↔문서 연결을 실제 터미널 이동 흐름처럼 다듬는다. 지도에서 문서를 열거나 문서에서 지도 요소를 열 때, 이전 권역/마커 선택 상태가 끊기지 않도록 복귀 경로를 보존한다.

## 변경 사항

### 1. 지도 → 문서 이동 시 지도 위치 기억
- 지도 요소 선택 상태에서 관련 기록을 열면 현재 권역, 필터, 검수 모드, 선택 요소를 저장한다.
- 내부 기록 보관서에 진입해도 직전 지도 위치를 잃지 않는다.

### 2. 문서 내부 `지도 복귀 경로` 추가
- 지도에서 문서를 연 경우, 해당 문서 상단에 `지도 복귀 경로` 블록을 표시한다.
- `지역지도 위치로 복귀` 버튼을 누르면 이전 권역과 선택 요소로 돌아간다.

### 3. 문서 → 지도 이동 시 이전 기록 기억
- 문서의 `관련 지도 기록` 버튼으로 지도 요소를 열면 지도 우측 패널에 `이전 기록으로 돌아가기` 버튼을 표시한다.
- 지도에서 확인 후 다시 원래 문서로 되돌아갈 수 있다.

### 4. 세계지도 리셋 완화
- 문서에서 지도 요소를 직접 열 때는 세계지도 첫 화면으로 리셋되지 않고 해당 권역/마커를 바로 연다.
- 일반 메뉴에서 지역지도를 새로 누르는 기본 동작은 기존처럼 권역 선택 화면을 유지한다.

### 5. SELF REVIEW 보강
- 자체 검수 리포트에 현재 이동 흐름 정보가 추가된다.
- `map-to-record`, `record-to-map` 컨텍스트와 대상 기록/지도 ID를 확인할 수 있다.

## 적용 기준

- 새 존 체계 추가 없음.
- 세계지도는 권역 선택 전용 구조 유지.
- R41 확장 요소 삭제 없음.
- R42 RECORD INDEX / SELF REVIEW / LINK AUDIT / 최종 점검 구조 유지.

- 관계 있는 요소는 가까이 두되, 정확히 겹치거나 선택이 어려운 배치는 피한다.

## 주요 수정

### 1. 확장 요소 위치 보정

항만권, 후방권, 해상 감시권, 도시 검문권에서 너무 가까웠던 일부 마커를 소폭 분리했다.

대상 예시:

- 동아시아: 홍콩 항만권, 부산·대한해협·동중국해 감시권
- 유럽: 피의 호수 주변, 북해 감시권, 코펜하겐 검문권
- 북미: 오대호 후방권, 밴쿠버 검역권
- 남아시아: 뉴델리 검문권, 뭄바이 후방권
- 동남아·인도양: 말라카·싱가포르 항로권, 보르네오 신호권
- 오세아니아: 호주 중앙 심부권, 애들레이드·태즈먼 후방권
- 중동: 바그다드 후방권, 수에즈·홍해 항로권
- 아프리카: 카이로 도시 검문권, 북아프리카 심부권, 동아프리카 해상권

### 2. 정확히 겹치던 요소 보정

오세아니아 중앙부에서 블랙존 후보지와 심부 반응점이 같은 좌표에 있던 상태를 보정했다.

- 블랙존은 중심 코어로 유지
- 사건 좌표는 블랙존 내부 인접 반응점으로 이동
- 후보 선택 메뉴 없이도 서로 구분 가능하도록 처리

### 3. 보조 마커 스케일 축소

R41에서 추가된 보조 사건·현상·시설·통신 마커 일부를 `small` 스케일로 조정했다.

의도:

- 핵심 존/봉쇄선보다 보조 항목이 과하게 튀지 않게 함
- 전체 필터에서 밀도가 높아진 권역의 가독성 유지
- 필터별 보기에서는 계속 충분히 식별 가능하게 유지

### 4. CSS 시각 무게 조정

- 보조 마커의 그림자와 선 두께 완화
- 작은 존의 관리 링/감시 링 투명도 완화
- 선택 시 비관계 요소 흐림 정도를 약간 완화
- 같은 관계권 요소는 더 안정적으로 남도록 조정

### 5. 검수 메타데이터 추가

`uac_region_map_data.js`에 `VISUAL_HOTFIX` 메타데이터를 추가했다. 릴리즈 후보 점검에서 R43 시각 보정 데이터가 로드되었는지 확인할 수 있다.

# PATCH 5.8.1R-44 — DetailMapZoomAndFocusPass

## 목적

R43의 확장 지도 시각 핫픽스와 149개 지도 요소를 유지하면서, 상세 권역지도에만 제한형 확대/초점 보기 기능을 추가한다.

세계지도는 계속 권역 선택 전용 화면으로 유지하며, 줌/드래그/FIT/WIDE 기능을 복구하지 않는다.

## 적용 내용

### 1. 상세지도 전용 보기 배율 추가

상세 권역지도 우측 패널에 `DETAIL VIEW` 컨트롤을 추가했다.

- 기본
- 확대
- 정밀

세계지도에서는 해당 컨트롤을 숨긴다.

### 2. 선택 요소 중심 보기

지도 요소를 선택한 상태에서 `선택 중심`을 누르면 선택 요소 주변을 기준으로 확대 표시한다.

### 3. 관계권 중심 보기

선택 요소가 관계권을 가진 경우 `관계권 중심`을 누르면 같은 관계권 요소 묶음을 기준으로 확대 표시한다.

예:

- lanzhou
- hongkong
- north-sea
- blood-lake
- central-aus

### 4. 확대/정밀 상태 라벨 강화

확대 상태에서는 선택 요소명이 표시되고, 정밀 + 관계권 중심 상태에서는 같은 관계권 요소명의 일부가 함께 표시된다.

검수 모드에서는 내부 ID와 관계권 코드도 함께 표시한다.

### 5. SELF REVIEW / 최종 점검 반영

검수 리포트와 SELF REVIEW 지표에 현재 상세 보기 상태를 추가했다.

- 보기 배율
- 초점 상태
- 선택 중심/관계권 중심 여부

### 6. API 추가

내부 검수 및 외부 호출용 API를 추가했다.

```text
ProjectCurseRegionalMap.setZoomMode(mode)
ProjectCurseRegionalMap.setZoomFocus(mode)
ProjectCurseRegionalMap.clearZoomFocus()
ProjectCurseRegionalMap.makeZoomFocusReport()
```
